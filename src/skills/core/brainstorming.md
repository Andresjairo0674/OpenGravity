# Skill: Brainstorming (Lluvia de Ideas)

Esta habilidad se activa automáticamente cuando el usuario solicita una nueva funcionalidad o un cambio complejo.

## Objetivo
Refinar el diseño y los requisitos antes de escribir una sola línea de código mediante técnicas socráticas.

## Cuándo usarla
- Al recibir una solicitud vaga.
- Al planear una nueva arquitectura o componente.
- Cuando hay múltiples formas de resolver un problema y la elección no es obvia.

## Instrucciones para el Agente
1. **NO programes inmediatamente**.
2. Haz de 2 a 3 preguntas críticas sobre el propósito, las restricciones y las preferencias del usuario.
3. Propón alternativas si las hay (ej: "podemos usar SQLite o Firebase, ¿cuál prefieres?").
4. Espera a la confirmación del diseño antes de proceder a la fase de Planning.
